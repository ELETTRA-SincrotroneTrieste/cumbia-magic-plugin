\doxysubsection{Detailed Description}
\begin{DoxyItemize}
\item Readings can be sequential or parallel (see the init method). Sequential readings must notify when a reading is performed and when a complete read cycle is over, providing the read data through two Qt signals\+: on\+New\+Data(const Cu\+Data\& da) and on\+Seq\+Read\+Complete(const Q\+List$<$\+Cu\+Data $>$\& data). Parallel readings must notify only when a new result is available, emitting the on\+New\+Data signal.\end{DoxyItemize}
\begin{DoxyItemize}
\item A multi reader must be initialised with the init method, that determines what is the engine used to read and whether the reading is sequential or parallel by means of the read\+\_\+mode parameter. If the mode is negative, the reading is parallel and the refresh mode is determined by the controls factory, as usual. If the mode is non negative {\itshape it must correspond to the {\bfseries{manual refresh mode}} of the underlying engine.} For example, Cu\+T\+Reader\+::\+Manual must be specified for the Tango control system engine in order to let the multi reader use an internal poller to read the attributes sequentially. \end{DoxyItemize}
